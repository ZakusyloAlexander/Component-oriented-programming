# Лабораторна робота 2 — Matching Emojis (хуки й основна логіка)

Ця версія застосунку продовжує каркас із лабораторної №1, але додає повноцінну гру з хуками для керування станом та логікою. Кожна сторінка використовує власні кастомні хуки, тож компоненти залишаються декларативними та сфокусованими на розмітці.

## Структура

- `index.html` — точка входу, підключає стилі та Vite-бандл.
- `styles/base.css` — загальні стилі та гріди.
- `src/main.js` — ініціалізація застосунку.
- `src/app.js` — рендер та навігація між сторінками.
- `src/state/appState.js` — центральне сховище стану.
- `src/hooks/` — набір кастомних хуків:
  - `useDifficulty.js` — вибір та збереження рівня складності.
  - `useGameSession.js` — логіка колоди, ходів та підрахунку статистики.
  - `useResults.js` — підготовка даних підсумків.
- `src/components/` — чисті компоненти:
  - `StartScreen.js`
  - `GameBoard.js`
  - `ResultsScreen.js`
- `package.json` — запуск через npm та Vite.

## Як запустити (npm + Vite)

```powershell
cd "C:\Учеба\компонентно оріентовне програмування\лаб 2"
npm install
npm run dev
```

Після запуску відкрийте посилання (типово `http://localhost:5173/`) і грайте.

## Особливості реалізації

- Колода генерується та тасується при кожному старті сесії, залежно від складності.
- Хуки інкапсулюють стани: вибір складності, логіку матчінгу карток та підрахунок статистики.
- Результати останньої гри зберігаються у глобальному стані і показуються на сторінці підсумків.
